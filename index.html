About The Project

Automated New Account Onboarding & Secure Credential Management

This project provides a robust platform for automating new account onboarding processes and ensuring secure credential management, particularly for SaaS platforms. It's designed with modern best practices for scalability, security, and maintainability.

Overview

The system intelligently processes onboarding requests, leverages AI for validation and decision-making, interacts securely with external services for account creation and credential storage, and aims for a high degree of automation and reliability. It serves as a central orchestration point, ensuring that all steps in the onboarding workflow are handled efficiently and securely.

Core Features

  • Automated Onboarding Request Detection: Monitors sources like Gmail for new onboarding triggers.
  • AI-Assisted Processing & Validation: Utilizes CrewAI for agentic intelligence, enabling complex reasoning, data extraction from requests, and decision-making.
  • Secure Account Creation & Credential Handling: Integrates with platform APIs for account provisioning and ensures credentials are never exposed insecurely.
  • Centralized Secrets Management: Leverages HashiCorp Vault (or similar) for secure storage and retrieval of API keys, passwords, and other sensitive credentials.
  • Modular & Extensible Architecture: Built with a clear separation of concerns, making it easy to integrate new platforms, agents, or tools.
  • FastAPI Backend: A high-performance asynchronous API backend serves as the central orchestrator.
  • Model Context Protocol (MCP): Employs a standardized data structure (MCP) for consistent context passing between services and agents, enhancing traceability and interoperability.

Architecture Overview

The system is architected around a central FastAPI backend that orchestrates workflows and interactions between various components:

[ Client UI / CLI ]
       |
       v
[  FastAPI Backend (Central Orchestrator) ]
    |      |     |        |
    v      v     v        v
[Gmail] [Vault] [Platform APIs] [CrewAI Agent(s) via MCP]

Client (UI/CLI): Initiates requests and interacts with the platform.

FastAPI Backend: The core orchestrator handling business logic, API requests, security, and integration with all subsystems.

CrewAI Layer: Provides agentic intelligence, performing tasks like analyzing requests, generating instructions, and making decisions using LLMs.

Model Context Protocol (MCP): Acts as the standardized data "envelope" for passing context, inputs, and results between FastAPI and CrewAI, and among CrewAI agents.

External Services:

  • Gmail: For detecting onboarding requests and sending notifications.
  • Vault/Secrets Manager: For secure storage and access to credentials.
  • Platform APIs: For performing actions like creating user accounts, assigning roles, etc.

Project Setup & Quickstart

Follow these steps to get the project up and running:

  1. Clone the Repository: git clone <your-repo-url>
  2. Set Up Virtual Environment: python3 -m venv venv && source venv/bin/activate
  3. Install Dependencies (Poetry recommended): pip install poetry && poetry install or pip install -r requirements.txt
  4. Configure Environment Variables: Copy .env.example to .env and populate.
  5. (Optional) Set Up External Services (Redis, Vault).

Running the Application & Testing

  1. Start FastAPI Backend: uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
  2. (If using Celery) Start Celery Worker(s): celery -A app.tasks.celery_app worker -l info -P eventlet
  3. (If using Celery Beat) Start Celery Beat: celery -A app.tasks.celery_app beat -l info
  4. Test Onboarding API: python scripts/trigger_onboarding_request.py ...
  5. Access API Docs: Swagger UI, ReDoc
